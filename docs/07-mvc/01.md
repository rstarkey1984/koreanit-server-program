# Spring Boot + Thymeleaf 소개

## 1. 개요

본 과정에서는 **Spring Boot와 Thymeleaf를 사용하여 관리자 페이지(Admin Page)** 를 구현한다.
Thymeleaf는 Spring Boot에서 공식적으로 지원하는 **서버 사이드 템플릿 엔진**으로,
서버에서 데이터를 HTML에 결합하여 **완성된 화면을 클라이언트에 전달**하는 방식이다.

---

## 2. Thymeleaf란 무엇인가

Thymeleaf는 **HTML 기반 템플릿 엔진**이다.

* HTML 문서를 그대로 사용한다
* 서버에서 데이터만 주입하여 화면을 완성한다
* 브라우저에서 HTML 단독으로 열어도 구조가 깨지지 않는다

즉,

> Thymeleaf는
> **HTML + 서버 데이터(Model)를 결합하여
> 서버에서 최종 HTML을 생성하는 기술**이다.

---

## 3. Spring Boot에서 Thymeleaf를 사용하는 이유

### 1) Spring MVC 구조 학습에 적합

요청 → Controller → Service → Repository → View 흐름이 명확하게 보인다.

```text
요청
 → Controller
 → Service
 → Repository
 → Model
 → Thymeleaf View
 → HTML 응답
```

---

### 2) 관리자 페이지에 최적화

관리자 페이지는 다음과 같은 특징을 가진다.

* 로그인 기반(세션)
* 화면 복잡도 낮음
* SEO 불필요
* 빠른 개발과 유지보수 중요

이 조건에 **Spring Boot + Thymeleaf 조합이 매우 적합**하다.

---

### 3) JSP 대체 기술

과거 Spring MVC에서는 JSP를 사용했으나,

* Java 코드 혼입
* 유지보수 어려움
* Spring Boot와 궁합이 좋지 않음

이러한 이유로 **현재는 Thymeleaf가 표준적인 대안**으로 사용된다.

---

## 4. Thymeleaf의 동작 방식

1. 클라이언트가 URL 요청
2. Controller가 데이터 조회
3. Model에 데이터 저장
4. Thymeleaf가 HTML에 데이터 치환
5. 완성된 HTML을 응답

```java
@GetMapping("/admin/users")
public String users(Model model) {
    model.addAttribute("users", userService.findAll());
    return "admin/users/list";
}
```

```html
<tr th:each="user : ${users}">
  <td th:text="${user.id}"></td>
  <td th:text="${user.username}"></td>
</tr>
```

---

## 5. 프로젝트에서의 역할 정리

| 구분          | 역할             |
| ----------- | -------------- |
| Spring Boot | 서버 로직, 비즈니스 처리 |
| Controller  | 요청 처리, View 선택 |
| Service     | 비즈니스 규칙        |
| Repository  | DB 접근          |
| Thymeleaf   | 서버 HTML 렌더링    |
| HTML        | 관리자 화면 UI      |

---
