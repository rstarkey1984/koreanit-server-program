# 서버프로그램 구현 과정 소개



## 1. 서버 프로그램이란 무엇인가

> 서버 프로그램은 **네트워크 요청을 처리하는 프로그램**이다.

서버 프로그램은 다음과 같은 특징을 가진다.

* 24시간 실행된다
* 동시에 여러 요청을 처리한다
* 네트워크를 통해 요청을 받고 응답을 보낸다

---

## 2. 서버 프로그램과 네트워크 · 포트

서버 프로그램은 **네트워크 상에서 특정 포트(port)를 열어두고**
요청을 기다린다.

### 네트워크 관점에서의 동작

```
클라이언트 (브라우저 / 앱)
   ↓  HTTP 요청 (IP + Port)
서버 IP : Port
   ↓
서버 프로그램
```

* **IP**: 서버가 존재하는 위치
* **Port**: 서버 프로그램이 대기 중인 통신 창구

### 포트의 의미

* 하나의 서버(IP)에는 여러 프로그램이 동시에 실행될 수 있다
* 각 프로그램은 **서로 다른 포트 번호**를 사용한다
* 포트를 통해 어떤 요청이 어떤 프로그램으로 들어갈지 구분한다

예시:

* `80` / `443` : 웹 서버(Nginx)
* `8080`, `9092` : API 서버(Spring Boot)
* `3306` : MySQL 데이터베이스

> 즉, **서버 프로그램 = 특정 포트를 열고 요청을 처리하는 프로그램**이다.

---

## 3. 이 강의에서 사용하는 환경

이 과정에서는 다음 환경을 사용한다.

### 운영체제 및 실행 환경

* Windows + WSL2
* Ubuntu 24.04 (리눅스 서버 환경)

### 서버 구성 요소

* Nginx (웹 서버, 외부 요청 수신)
* Spring Boot (WAS, 실제 서버 프로그램)
* MySQL 8.x (데이터베이스 서버)

### 개발 및 관리 도구

* Git (형상관리)
* 터미널 기반 작업
* VSCode는 코드 편집 및 터미널 작업 도구로 사용한다

  - VSCode 설치 - `https://code.visualstudio.com/download`
  
  - 확장 설치 - `Korean Language Pack for Visual Studio Code`
