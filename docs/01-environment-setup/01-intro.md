# 서버프로그램 구현 과정 소개

## 강의 목표

이 과정을 통해 학생은 다음을 이해하고 설명할 수 있어야 한다.

* 서버 프로그램이 무엇인지 설명할 수 있다
* 웹 서비스에서 서버의 역할을 이해한다
* 왜 서버 프로그램은 리눅스 환경을 기준으로 개발하는지 이해한다
* 이 강의에서 어떤 환경과 기술을 사용하는지 전체 흐름을 파악한다

---

## 1. 서버 프로그램이란 무엇인가

우리가 만드는 프로그램은 **사용자 PC에서 실행되는 프로그램**이 아니다.

서버 프로그램은 다음과 같은 특징을 가진다.

* 화면(UI)이 없다
* 24시간 실행된다
* 동시에 여러 요청을 처리한다
* 네트워크를 통해 요청을 받고 응답을 보낸다

예를 들면 다음과 같다.

* 로그인 요청 처리 (아이디/비밀번호 검증)
* 게시글 목록 조회 API
* 결제 요청 처리
* 뉴스 데이터 수집 배치 프로그램

이 모든 것은 **서버에서 실행되는 프로그램**이다.

---

## 2. 서버 프로그램과 클라이언트 프로그램의 차이

### 클라이언트 프로그램

* 사용자가 직접 실행
* 화면이 존재
* 사용자 입력 중심
* 예: 웹 브라우저, 모바일 앱, 데스크톱 프로그램

### 서버 프로그램

* 서버에서 항상 실행 중
* 화면이 없음
* 요청(Request)과 응답(Response) 중심
* 예: 웹 서버, API 서버, 배치 프로그램

이 과정에서는 **서버 프로그램**에 집중한다.

---

## 3. 서버 프로그램은 어떻게 동작하는가

기본 흐름은 다음과 같다.

```
클라이언트 요청
   ↓
서버 프로그램
   ↓
DB / 파일 / 외부 API
   ↓
응답 반환
```

서버 프로그램의 핵심 역할은 다음 세 가지다.

1. 요청을 정확히 해석한다
2. 필요한 처리를 수행한다
3. 결과를 빠르고 안정적으로 응답한다

---

## 4. 이 강의의 핵심 방향

이 과정은 단순히 코드를 따라 치는 강의가 아니다.

다음 구조를 **명확히 이해하는 것**이 목표다.

```
HTTP 요청
 → Controller
 → Service
 → Repository(SQL)
 → Database
```

각 계층의 역할은 다음과 같다.

* Controller
  요청과 응답을 담당
* Service
  비즈니스 로직과 흐름 제어
* Repository
  SQL 실행과 DB 접근
* Database
  데이터 저장

이 구조는 실무 서버 프로그램의 기본 형태다.

---

## 5. 왜 서버 프로그램 환경 구성이 중요한가

서버 프로그램은 코드만 잘 짜면 끝이 아니다.

* 어떤 OS에서 실행되는가
* 어떤 서버 환경을 기준으로 개발하는가
* 실제 운영 환경과 얼마나 비슷한가

이것들이 매우 중요하다.

그래서 이 강의는 처음부터 **서버 환경 구성**부터 시작한다.

---

## 6. 이 강의에서 사용하는 환경

이 과정에서는 다음 환경을 사용한다.

### 운영체제 및 실행 환경

* Windows + WSL2
* Ubuntu 24.04 (리눅스 서버 환경)

### 서버 구성 요소

* Nginx (웹 서버)
* Spring Boot (WAS)
* MySQL 8.x (데이터베이스)

### 개발 및 관리 도구

* Git (형상관리)
* 터미널 기반 작업
* VSCode는 코드 편집 및 터미널 작업 도구로 사용한다

이 환경은 **실제 서버 운영 환경과 최대한 유사**하게 구성된다.

---

## 7. 이 과정을 마치면 할 수 있는 것

이 과정을 끝내면 다음을 할 수 있어야 한다.

* 리눅스 서버 환경에서 작업할 수 있다
* 서버 프로그램의 전체 흐름을 이해한다
* API 서버 구조를 설명할 수 있다
* 실무에서 사용하는 서버 환경이 낯설지 않다
