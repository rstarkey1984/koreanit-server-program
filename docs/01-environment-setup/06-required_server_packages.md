# 필수 서버 패키지 설치

*(curl, git, build-essential)*

이 장에서는
Ubuntu 서버에서 **앞으로 계속 사용하게 될 기본 도구들**을 설치한다.

학생은 이 장을 끝내면
“서버에 기본으로 깔아두는 도구 세트”가 무엇인지 이해해야 한다.

---

## 강의 목표

* 서버에서 패키지를 설치하는 기본 흐름을 이해한다
* 서버 초기 세팅 시 update / upgrade의 의미를 이해한다
* curl, git, build-essential의 역할을 설명할 수 있다
* 이후 모든 실습 환경을 안정적으로 준비한다

---

## 1. 서버에서 패키지 설치란?

리눅스 서버에서는
프로그램을 **웹에서 직접 다운로드하지 않는다**.

대신
운영체제가 관리하는 **패키지 관리자(apt)** 를 사용한다.

* 중앙 저장소에서 검증된 프로그램 설치
* 의존성 자동 해결
* 업데이트 관리 가능

→ **서버 환경의 표준 방식**

---

## 2. 필수 서버 패키지 목록

이번 단계에서 설치할 패키지는 다음 3가지다.

### 2-1. curl

* HTTP 요청 테스트 도구
* API 서버 상태 확인
* 서버 간 통신 테스트에 사용

예:

```bash
curl https://example.com
```

---

### 2-2. git

* 소스 코드 형상 관리 도구
* GitHub / GitLab 연동 필수
* 서버 배포 및 협업의 기본

---

### 2-3. build-essential

* C/C++ 컴파일 도구 모음
* 내부 구성:

  * gcc
  * make
  * libc-dev

> 많은 서버 프로그램이
> 내부적으로 빌드 도구를 요구한다.

---

## 3. 서버 초기 업데이트 (중요)

서버를 처음 세팅할 때는
패키지를 설치하기 전에 반드시
**패키지 목록 갱신과 업그레이드**를 먼저 수행한다.

```bash
sudo apt update
sudo apt upgrade -y
```

* `update` : 최신 패키지 목록 갱신
* `upgrade` : 설치된 패키지 최신 반영

> 이 단계는 한 번만 해두면 되며,
> 서버 보안과 안정성에 직접적인 영향을 준다.

---

## 4. 필수 패키지 설치

```bash
sudo apt install -y curl git build-essential
```

* `-y` : 설치 중 질문 자동 승인

---

## 5. 설치 확인

### 5-1. curl 확인

```bash
curl --version
```

---

### 5-2. git 확인

```bash
git --version
```

---

### 5-3. build-essential 확인

```bash
gcc --version
make --version
```

---

## 이 장의 핵심 정리

* 서버 프로그램은 패키지 관리자로 설치한다
* 서버 초기 세팅 시 반드시 update → upgrade를 먼저 수행한다
* curl : 서버 통신 테스트 도구
* git : 코드 관리 및 배포 도구
* build-essential : 컴파일 기반 도구
