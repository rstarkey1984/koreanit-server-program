# Ubuntu 서버 기본 사용법

이 장에서는
이미 설치된 Ubuntu 서버 환경에서
**서버를 직접 사용하며 기본 조작에 익숙해지는 것** 을 목표로 한다.

명령어를 많이 외우는 것이 아니라,
“지금 내가 서버의 어느 경로에 있고, 무엇을 하고 있는지”를 이해하는 단계다.

---

## 1. 서버에 접속해 있다는 의미

SSH로 Ubuntu 서버에 접속하면 다음과 같은 프롬프트를 보게 된다.

```bash
ubuntu@ubuntu24:~$
```

각 요소의 의미는 다음과 같다.

* `ubuntu` : 로그인한 사용자 이름
* `ubuntu24` : 서버 이름(hostname)
* `~` : 홈 디렉터리
* `$` : 일반 사용자 권한

> 이 프롬프트가 보인다는 것은
> **내 PC가 아니라 서버 안에서 명령을 실행하고 있다는 뜻**이다.

---

## 2. 현재 위치 확인하기

```bash
pwd
```

* 현재 작업 중인 디렉터리 경로를 출력한다
* 서버 작업 중 길을 잃었을 때 가장 먼저 사용하는 명령이다

---

## 3. 파일과 디렉터리 목록 확인
```bash
ls
```
숨김 파일을 포함해 파일 정보를 자세히 출력하는 명령어:
```bash
ls -al
ll
```


---

## 4. 디렉터리 이동하기

```bash
cd /
cd ~
cd /home/ubuntu
```

의미:

* `/` : 서버의 최상위 디렉터리
* `~` : 현재 사용자의 홈 디렉터리

> 서버 작업은 보통 **홈 디렉터리를 기준으로 시작**한다.

---

## 5. 파일 직접 만들어보기

빈 파일 생성:

```bash
touch hello.txt
ls
```

파일 내용 작성:

```bash
nano hello.txt
```

예시 내용:

```text
Hello Ubuntu Server
```

저장: `Ctrl + O`
종료: `Ctrl + X`

---

## 6. 현재 사용자 확인

```bash
whoami
```

* 현재 로그인한 사용자 이름을 출력한다
* 권한 오류가 발생했을 때 가장 먼저 확인해야 한다

---

## 7. sudo의 의미 (개념 이해)

Ubuntu 서버에서는
일부 작업에 **관리자 권한**이 필요하다.

```bash
sudo 명령어
```

의미:

> “이 명령을 관리자 권한으로 실행한다”

패키지 설치, 시스템 설정 변경, 서비스 제어 등은
`sudo` 없이는 실행할 수 없다.
