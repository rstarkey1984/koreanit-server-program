# Ubuntu 서버 맛보기


## 1. 서버에 접속해 있다는 의미

VS Code 하단 패널 설정

![vscode 하단 패널](/img/bottom_pannel.png?1)

SSH로 Ubuntu 서버에 접속하면 다음과 같은 프롬프트를 보게 된다.

```bash
ubuntu@ubuntu24:~$
```

각 요소의 의미는 다음과 같다.

* `ubuntu` : 로그인한 사용자 이름
* `ubuntu24` : 서버 이름(hostname)
* `~` : 홈 디렉터리
* `$` : 일반 사용자 권한

> 이 프롬프트가 보인다는 것은  
> **명령이 내 PC가 아니라 서버(Ubuntu) 안에서 실행된다는 뜻** 이다.

---

## 2. 현재 위치 확인하기

```bash
pwd
```

* 현재 작업 중인 디렉터리 경로를 출력한다
* 서버 작업 중 길을 잃었을 때 가장 먼저 사용하는 명령이다

---

## 3. 파일과 디렉터리 목록 확인
```bash
ls
```
숨김 파일을 포함해 파일 정보를 자세히 출력하는 명령어:
```bash
ls -al
ll
```


---

## 4. 디렉터리 이동하기

```bash
cd /
cd ~
cd /home/ubuntu
```

의미:

* `/` : 서버의 최상위 디렉터리
* `~` : 현재 사용자의 홈 디렉터리

> 서버 작업은 보통 **홈 디렉터리를 기준으로 시작** 한다.

---

## 5. 파일 생성 및 작성

빈 파일 생성:

```bash
touch hello.txt
ls
```

파일 내용 작성:

```bash
nano hello.txt
```

예시 내용:

```text
Hello Ubuntu Server
```
종료: `Ctrl + X` 

파일내용 변경 있을시
  - `Y` 키 입력 → 엔터 : 저장
  - `N` 키 입력 : 저장하지 않고 종료

## 6. 파일 내용 확인하기 (cat)

`cat`은 파일의 내용을 **그대로 화면에 출력** 하는 명령어다.

```bash
cat hello.txt
```

출력 예:

```text
Hello Ubuntu Server
```

* 파일을 수정하지 않고 내용만 확인
* 짧은 파일 확인에 사용

> `cat`은 파일 내용을 빠르게 확인하는 명령어다.

---

## 7. 로그 끝부분 확인하기 (tail)

`tail`은 로그 파일의 **마지막 부분을 확인** 하는 명령어다.

예시 (시스템 로그):

```bash
tail /var/log/syslog
```

* 기본적으로 마지막 10줄 출력

실시간 로그 확인:

```bash
tail -f /var/log/syslog
```

* 새 로그가 추가되면 즉시 출력
* 서버 동작 상태 확인에 사용

> `tail`은 실행 중인 서버 로그를 실시간으로 볼 때 사용한다.
