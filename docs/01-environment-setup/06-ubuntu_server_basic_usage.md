# Ubuntu 서버 기본 사용법

이 장에서는
이미 설치된 Ubuntu 서버 환경에서
**서버를 직접 사용하며 기본 조작에 익숙해지는 것** 을 목표로 한다.

명령어를 많이 외우는 것이 아니라,
“지금 내가 서버의 어느 경로에 있고, 무엇을 하고 있는지”를 이해하는 단계다.

---

## 1. 서버에 접속해 있다는 의미

VS Code 하단 패널 설정

![vscode 하단 패널](/img/bottom_pannel.png?1)

SSH로 Ubuntu 서버에 접속하면 다음과 같은 프롬프트를 보게 된다.

```bash
ubuntu@ubuntu24:~$
```

각 요소의 의미는 다음과 같다.

* `ubuntu` : 로그인한 사용자 이름
* `ubuntu24` : 서버 이름(hostname)
* `~` : 홈 디렉터리
* `$` : 일반 사용자 권한

> 이 프롬프트가 보인다는 것은  
> **명령이 내 PC가 아니라 서버(Ubuntu) 안에서 실행된다는 뜻**이다.

---

## 2. 현재 위치 확인하기

```bash
pwd
```

* 현재 작업 중인 디렉터리 경로를 출력한다
* 서버 작업 중 길을 잃었을 때 가장 먼저 사용하는 명령이다

---

## 3. 파일과 디렉터리 목록 확인
```bash
ls
```
숨김 파일을 포함해 파일 정보를 자세히 출력하는 명령어:
```bash
ls -al
ll
```


---

## 4. 디렉터리 이동하기

```bash
cd /
cd ~
cd /home/ubuntu
```

의미:

* `/` : 서버의 최상위 디렉터리
* `~` : 현재 사용자의 홈 디렉터리

> 서버 작업은 보통 **홈 디렉터리를 기준으로 시작**한다.

---

## 5. 파일 생성 및 작성

빈 파일 생성:

```bash
touch hello.txt
ls
```

파일 내용 작성:

```bash
nano hello.txt
```

예시 내용:

```text
Hello Ubuntu Server
```
종료: `Ctrl + X` 

파일내용 변경 있을시
  - `Y` 키 입력 → 엔터 : 저장
  - `N` 키 입력 : 저장하지 않고 종료

## 6. 현재 사용자 확인

```bash
whoami
```

* 현재 로그인한 사용자 이름을 출력한다
* 권한 오류가 발생했을 때 가장 먼저 확인해야 한다

---

## 7. 파일 내용 확인하기 (cat)

`cat`은 파일의 내용을 **그대로 화면에 출력**하는 명령어다.

```bash
cat hello.txt
```

출력 예:

```text
Hello Ubuntu Server
```

* 파일을 수정하지 않고 내용만 확인
* 짧은 파일 확인에 사용

> `cat`은 파일 내용을 빠르게 확인하는 명령어다.

---

## 8. 로그 끝부분 확인하기 (tail)

`tail`은 로그 파일의 **마지막 부분을 확인**하는 명령어다.

예시 (시스템 로그):

```bash
tail /var/log/syslog
```

* 기본적으로 마지막 10줄 출력

실시간 로그 확인:

```bash
tail -f /var/log/syslog
```

* 새 로그가 추가되면 즉시 출력
* 서버 동작 상태 확인에 사용

> `tail`은 실행 중인 서버 로그를 실시간으로 볼 때 사용한다.


---

## sudo의 의미 (개념 이해)

Ubuntu 서버에서는
일부 작업에 **관리자 권한**이 필요하다.

```bash
sudo 명령어
```

의미:

> “이 명령을 관리자 권한으로 실행한다”

패키지 설치, 시스템 설정 변경, 서비스 제어 등은
`sudo` 없이는 실행할 수 없다.
