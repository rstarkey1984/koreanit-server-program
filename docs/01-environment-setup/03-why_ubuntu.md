# 03. Ubuntu를 사용하는가

## 강의 목표

이 장이 끝나면 학생은 다음을 이해하고 설명할 수 있어야 한다.

* 리눅스 배포판이 무엇인지 설명할 수 있다
* 서버 환경에서 배포판 선택이 왜 중요한지 이해한다
* 여러 리눅스 배포판 중 Ubuntu를 선택하는 이유를 설명할 수 있다

---

## 1. 리눅스는 하나가 아니다

앞 장에서 서버 프로그램은 리눅스 환경에서 운영된다는 것을 배웠다.
하지만 리눅스는 하나의 OS 이름이 아니라,
**여러 배포판(distribution)** 으로 나뉘어 있다.

즉, 리눅스는 다음과 같은 구조를 가진다.

* 공통된 리눅스 커널
* 배포판마다 다른 패키지 구성과 정책

그래서 서버를 만들 때는
"리눅스를 쓴다"가 아니라
**"어떤 리눅스 배포판을 쓸 것인가"** 를 선택해야 한다.

---

## 2. 서버에서 배포판 선택이 중요한 이유

리눅스 배포판마다 다음 요소가 다르다.

* 기본 패키지 구성
* 패키지 관리자
* 업데이트 정책
* 보안 패치 주기
* 문서와 자료의 양

서버 환경에서는 **"가장 안정적이고 자료가 많은 배포판"** 이 중요하다.

---

## 3. 대표적인 리눅스 서버 배포판

서버 환경에서 자주 언급되는 배포판은 다음과 같다.

* Ubuntu Server
* Debian
* CentOS / Rocky Linux / AlmaLinux
* Amazon Linux

이들 모두 서버로 사용할 수 있지만,
**교육과 실습 환경에서는 차이가 크다.**

---

## 4. Ubuntu가 많이 선택되는 이유

Ubuntu는 서버 환경에서 다음과 같은 장점을 가진다.

### 4-1. 사용자가 많고 자료가 압도적으로 많다

* 검색했을 때 해결 방법이 잘 나온다
* 한글 자료도 비교적 풍부하다
* 초보자가 막히는 지점에 대한 정보가 많다

서버 환경에서 자료의 양은
**문제 해결 속도와 직결된다.**

---

### 4-2. 표준에 가까운 설정 방식

Ubuntu의 설정 방식은
특정 회사나 환경에 종속되지 않는다.

* 디렉터리 구조가 표준적이다
* 설정 파일 위치가 예측 가능하다
* 다른 리눅스로 옮겨가도 응용이 쉽다

즉,
Ubuntu에서 배운 내용은
**다른 리눅스 서버에서도 그대로 활용 가능**하다.

---

### 4-3. 서버와 데스크톱 환경의 일관성

Ubuntu는
서버와 데스크톱 환경 모두를 제공한다.

* 집에서 연습하기 쉽다
* 개인 PC에서도 동일한 OS 사용 가능
* 실습 환경 재현이 쉽다

이 점은 교육 환경에서 매우 큰 장점이다.

---

### 4-4. 장기 지원(LTS) 정책

Ubuntu는
**LTS(Long Term Support)** 버전을 제공한다.

* 장기간 보안 업데이트 제공
* 서버 운영에 적합
* 잦은 버전 변경 부담이 적다

이 과정에서는
**Ubuntu 24.04 LTS**를 사용한다.

---

## 5. 왜 이 강의에서 Ubuntu를 사용하는가

이 강의의 목적은
리눅스 자체를 깊게 배우는 것이 아니다.

목표는 다음과 같다.

* 서버 프로그램 구조 이해
* 실제 서버와 유사한 환경 경험
* 실무에서 쓰는 흐름 익히기

이 목적에 가장 잘 맞는 선택이
**Ubuntu**다.

---

## 정리

* 리눅스에는 여러 배포판이 있다
* 서버 환경에서는 안정성과 자료의 양이 중요하다
* Ubuntu는 서버·교육 환경 모두에 적합하다
* 이 과정에서는 Ubuntu 24.04 LTS를 사용한다
