# 왜 서버 프로그램은 리눅스를 사용하는가

## 강의 목표

이 장이 끝나면 학생은 다음을 설명할 수 있어야 한다.

* 서버 프로그램과 운영체제(OS)의 관계를 이해한다
* 왜 서버 환경에서 리눅스가 표준처럼 사용되는지 설명할 수 있다
* GUI 중심 OS와 서버용 OS의 차이를 구분할 수 있다

---

## 1. 서버 프로그램은 어떤 환경에서 실행되는가

서버 프로그램은 다음과 같은 환경에서 실행된다.

* 24시간 꺼지지 않고 실행됨
* 동시에 많은 요청을 처리함
* 사람이 직접 조작하지 않음
* 장애가 나면 빠르게 복구되어야 함

즉, 서버 프로그램은 **사람을 위한 환경이 아니라 시스템을 위한 환경**에서 실행된다.

---

## 2. 운영체제의 역할 다시 생각하기

운영체제(OS)는 단순히 화면을 보여주는 프로그램이 아니다.

운영체제의 핵심 역할은 다음과 같다.

* CPU 사용 관리
* 메모리 관리
* 파일 시스템 관리
* 프로세스 실행 및 종료 관리
* 네트워크 처리

서버 환경에서는 이 기능들이 **안정적이고 예측 가능하게** 동작하는 것이 가장 중요하다.

---

## 3. GUI 중심 OS가 서버에 적합하지 않은 이유

GUI 중심 OS는 기본 전제가 다르다.

* 사람이 직접 로그인한다
* 화면을 보며 마우스로 조작한다
* 사용자 편의성이 중요하다

하지만 서버 환경은 다르다.

* 화면이 필요 없다
* 마우스를 쓰지 않는다
* 자동 실행과 무인 운영이 중요하다

GUI는 서버 입장에서 보면 다음과 같은 단점을 가진다.

* 불필요한 프로세스가 상시 실행됨
* CPU·메모리 자원 낭비
* 자동화가 어렵다
* 보안 관리 대상이 늘어난다

서버는 **필요한 기능만 최소로 실행하는 환경**이 가장 안정적이다.

---

## 4. 리눅스가 서버 환경에 적합한 이유

리눅스는 처음부터 서버 환경을 염두에 두고 발전해 왔다.

리눅스의 주요 특징은 다음과 같다.

* GUI 없이도 완전한 운영 가능
* 텍스트 기반(CLI) 환경
* 자동화 및 스크립트 실행에 최적화
* 안정적인 장기 실행
* 서버 운영에 불필요한 요소 제거 가능

그래서 실무에서는 자연스럽게 이런 선택을 한다.

> 서버 프로그램은 리눅스 기준으로 개발한다

---

## 5. 실제 서버 운영 방식

실제 서버 운영은 다음과 같은 방식으로 이루어진다.

* SSH로 서버 접속
* 명령어로 서비스 제어
* 스크립트로 배포 및 실행
* 재부팅 시 자동 실행 설정

서버는 **사람이 계속 붙어서 관리하는 대상이 아니다**.

리눅스는 이런 운영 방식에 가장 잘 맞는 운영체제다.

---

## 6. 개발 환경과 운영 환경을 맞추는 이유

서버 프로그램 개발에서 중요한 원칙이 있다.

> 개발 환경과 운영 환경은 최대한 같아야 한다

운영 서버가 리눅스라면,
개발 환경도 리눅스를 기준으로 맞추는 것이 가장 안전하다.

그래서 이 과정에서는 Windows 위에서 바로 개발하지 않고,
**리눅스 서버 환경을 먼저 구성**한다.

---

## 정리

* 서버 프로그램은 시스템 중심 환경에서 실행된다
* GUI 중심 OS는 서버 환경에 불리하다
* 리눅스는 서버 운영에 최적화된 운영체제다
* 서버 프로그램은 리눅스 기준으로 개발하는 것이 표준이다
