# 왜 서버 프로그램은 리눅스를 사용하는가 

## 1. 서버 프로그램이 요구하는 환경

서버 프로그램은 다음 조건을 전제로 실행된다.

* 24시간 중단 없이 실행
* 다수의 동시 요청 처리
* 무인 운영
* 장애 발생 시 빠른 복구 필요

→ 서버 프로그램은 **사람 중심이 아닌 시스템 중심 환경**에서 동작한다.

---

## 2. 서버 환경에서 운영체제(OS)의 역할

운영체제는 화면을 보여주는 도구가 아니라
**시스템 자원을 관리하는 핵심 소프트웨어**다.

서버 환경에서 중요한 OS 역할:

* CPU 관리
* 메모리 관리
* 프로세스 관리
* 파일 시스템 관리
* 네트워크 처리

→ 서버에서는 기능이 **안정적이고 예측 가능하게 동작하는 것**이 가장 중요하다.

---

## 3. GUI 중심 OS가 서버 환경에 부적합한 이유

GUI 중심 OS 특징:

* 사용자 직접 조작 전제
* 화면 기반 인터페이스
* 사용자 편의성 중심

서버 환경에서는 다음과 같은 문제가 발생한다.

* 불필요한 프로세스 상시 실행
* CPU·메모리 자원 낭비
* 자동화·원격 관리에 불리
* 보안 관리 대상 증가

→ 서버 환경에서는 **GUI가 필수가 아니다**.

---

## 4. 리눅스가 서버에 적합한 이유

리눅스의 서버 환경 특징:

* GUI 없이 운영 가능
* CLI (Command Line Interface) 기반 관리
* 스크립트·자동화에 적합
* 장기 실행에 안정적
* 필요한 기능만 최소 구성 가능

→ 서버 운영에 필요한 조건을 구조적으로 만족한다.

---

## 5. 개발 환경과 운영 환경의 일치

서버 프로그램 개발의 기본 원칙:

> 개발 환경과 운영 환경은 최대한 같아야 한다

* 실제 운영 서버는 대부분 리눅스
* 개발 환경도 리눅스를 기준으로 구성해야 오류가 줄어든다

→ 서버 프로그램은 **리눅스 기준으로 개발하는 것이 표준**이다.

---

### 한줄 요약 

> 서버 프로그램은 무인·자동화·장기 운영에 적합한 CLI 기반 리눅스를 표준 운영체제로 사용한다.
