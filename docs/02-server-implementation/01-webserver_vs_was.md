# 웹 서버와 WAS 개념 이해

이 장에서는
**웹 서버(Web Server)** 와 **WAS(Web Application Server)** 의 역할을 명확히 구분한다.

지금까지 설치한 Nginx는
"서버 프로그램"이 아니라
**서버 프로그램 앞단에서 요청을 처리하는 웹 서버**라는 점을 정확히 이해하는 것이 목표다.


---

## 1. 서버는 하나가 아니다

초보자에게 서버는 보통 이렇게 보인다.

> "요청 오면 처리해서 응답하는 프로그램"

하지만 실제 서버 구조는 **역할이 분리**되어 있다.

```text
[Browser]
   ↓
[Web Server]
   ↓
[WAS]
   ↓
[Database]
```

이 구조를 이해하지 못하면
이후 모든 서버 개념이 흐릿해진다.

---

## 2. 웹 서버(Web Server)의 역할

웹 서버는 **요청을 가장 먼저 받는 서버**다.

대표적인 역할:

* HTTP 요청 수신
* 정적 파일 제공 (HTML, CSS, JS, 이미지)
* 요청 전달(프록시)
* SSL 처리

지금까지 사용한 **Nginx**가 바로 웹 서버다.

중요 포인트:

> 웹 서버는 **비즈니스 로직을 처리하지 않는다**.

---

## 3. WAS(Web Application Server)의 역할

WAS는
**서버 프로그램이 실제로 실행되는 영역**이다.

역할:

* 비즈니스 로직 처리
* Controller / Service / Repository 실행
* DB 연동
* 동적 응답 생성

이 강의에서 WAS는 **Spring Boot**다.

---

## 4. 왜 웹 서버와 WAS를 분리할까?

### 4-1. 역할 분리

* 웹 서버: 요청 처리, 전달, 보호
* WAS: 로직 처리

각자 잘하는 것만 한다.

---

### 4-2. 성능과 안정성

* 정적 파일은 웹 서버가 처리
* WAS는 로직에 집중
* 장애 격리 가능

---

### 4-3. 확장성

```text
[Web Server]
   ↓
[WAS 1]
[WAS 2]
[WAS 3]
```

웹 서버 앞단에서
여러 WAS로 요청을 분산할 수 있다.

---

## 5. 지금까지 우리가 한 일 다시 보기

| 단계        | 의미         |
| --------- | ---------- |
| Ubuntu 설치 | 서버 OS 준비   |
| Nginx 설치  | 웹 서버 준비    |
| 정적 파일 확인  | 웹 서버 역할 체감 |
| Git 사용    | 서버 작업 관리   |

아직 **서버 프로그램은 하나도 만들지 않았다**.

---

## 6. 이제부터 할 일

이제 해야 할 일은 명확하다.

> 웹 서버 뒤에서 동작하는
> **서버 프로그램(WAS)** 을 직접 만드는 것

그게 바로 다음 장이다.

---

## 이 장의 핵심 정리

* 웹 서버와 WAS는 역할이 다르다
* Nginx는 웹 서버다
* Spring Boot는 WAS다
* 서버 구조는 반드시 분리해서 생각해야 한다
