# 04. 배치 프로그램 구현하기 (Node.js 워커)

이 파트에서는  
**HTTP 요청을 받지 않는 독립 실행형 배치/워커 프로그램**을 Node.js로 구현한다.

메인 서버(Spring Boot)는 요청/응답 처리만 담당하고,  
배치 프로그램은 **정해진 작업을 수행한 뒤 종료되는 별도의 프로세스**로 구성한다.

---

## 이 파트의 목표

* 배치/워커 프로그램의 개념을 이해한다
* Node.js로 실행형(one-shot) 워커를 구현한다
* 메인 서버와 워커를 프로세스 단위로 분리하는 이유를 이해한다

## 학습 결과

* 요청이 없는 프로그램 구조를 설명할 수 있다
* 워커의 실행 흐름을 코드 기준으로 설명할 수 있다
* 서버와 워커를 역할 기준으로 분리할 수 있다

---

## 학습 포인트

* 워커는 서버가 아니다
* 실행 → 작업 → 종료 구조를 가진다
* 성공/실패는 로그가 아니라 종료 코드로 판단한다
