# FULL_SCAFFOLD – 서버프로그램 강의안 구조 요약

이 문서는
지금까지 설계·작성한 **서버프로그램 강의안의 전체 구조, 규칙, 진행 방식**을
다른 컨텍스트에서도 그대로 이어가기 위한 기준 요약본이다.

---

## 1. 전체 파트 구성

```text
docs/
├── 01-environment-setup/
├── 02-server-implementation/
├── 03-common-modules/
├── 04-batch-program/
└── (이후 확장)
```

각 파트는 **README.md + 단계별 md 파일** 구조를 가진다.

---

## 2. 공통 설계 철학 (가장 중요)

* 문법보다 구조
* 구현보다 흐름
* 기능보다 책임 분리
* “돌아가는 코드”보다 “설명 가능한 구조”

강의안은 **프레임워크 튜토리얼이 아니라 구조 교육용**이다.

---

## 3. 파일 / 폴더 네이밍 규칙

### 폴더

* `kebab-case`
* 파트 단위로 구분

### 파일

* `NN-description.md`
* 번호는 README에서만 흐름 의미
* 문서 제목에는 번호를 붙이지 않음

예:

```text
01-common_response_format.md
03-global_exception_handler.md
```

---

## 4. README.md 공통 규칙

각 파트의 README.md는 다음 구조를 따른다.

1. 파트 제목
2. 파트 설명 (왜 필요한가)
3. 이 파트에서 다루는 것
4. 문서 구성 (번호 + 파일명)
5. 이 파트의 학습 결과
6. 학습 포인트 (철학/강조점)
7. 강의 시작 링크

README는 **목차가 아니라 진입 페이지**다.

---

## 5. 01-environment-setup

**목적**

* 개발 환경이 아닌 서버 환경 감각 형성

**특징**

* Linux / Ubuntu 선택 이유
* WSL2 + Ubuntu 서버 구성
* 서버 기본 세팅
* 필수 패키지
* 리눅스 서버 기초 감각

구현 코드는 거의 포함하지 않는다.

---

## 6. 02-server-implementation

**목적**

* 서버 내부 구조와 요청 흐름 이해

**핵심 내용**

* Web Server vs WAS
* Controller / Service / Repository
* MySQL / JDBC / 트랜잭션
* 요청 → 처리 → 응답 흐름
* CRUD 구조
* 에러 / 로깅 / 리팩토링까지 한 사이클

여기까지가 **서버 기본기**다.

---

## 7. 03-common-modules

**성격**

* 서버 전반에서 쓰이는 규칙과 표준 정립

**핵심 원칙**

* 개념 설명 후 즉시 실습
* 실습 파일 분리하지 않음
* 각 이론 문서 하단에 `## 실습` 섹션 추가

**구성**

```text
03-common-modules/
├── README.md
├── 01-common_response_format.md
├── 02-error_codes_and_exceptions.md
├── 03-global_exception_handler.md
├── 04-request_logging.md
├── 05-profiles_and_config.md
```

---

## 8. 공통 모듈 실습 방식

* 실습 전용 폴더 사용하지 않음
* 실습 전용 파일 사용하지 않음
* 이론 문서 마지막에 실습 추가

**목표**

* 새로운 개념 학습 ❌
* 합의된 규칙을 코드로 고정 ⭕

---

## 9. 공통 모듈 구현 기준

* JPA 사용 안 함
* Spring JDBC 기반
* Controller:

  * try-catch 사용 ❌
  * 응답 포맷 직접 생성 ❌
* 예외:

  * Service/Repository에서 발생
  * Global Handler에서 응답 변환
* 로깅:

  * Controller 외부
  * 최소 정보만 기록

---

## 10. 04-batch-program 방향

04 파트는 **공통 모듈 구현 파트가 아니다**.

**목적**

* 요청이 없는 프로그램 구조 이해
* 스케줄 / 작업 중심 처리
* 공통 모듈의 실전 활용

03에서 만든 규칙을 **강제로 사용하게 되는 환경**이다.

---

## 11. 강의 운영 원칙

* 강의자 메모는 학생과 공유하지 않는다
* 이론 → 즉시 실습 리듬 유지
* “더 좋은 방법” 논쟁 차단
* 지금 단계에서 하지 않는 것 명확히 선언
